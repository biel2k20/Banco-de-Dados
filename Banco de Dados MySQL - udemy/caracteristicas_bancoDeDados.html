Caractersticas de um banco de dados

Bancos de dados devem executar procedimentos chamados de transaes. A integridade de uma transao deve ser regida por quatro propriedades:

  • Atomicidade

    • Todas as aes devem ser concludas com sucesso, ou o processo falha como um todo e toda a ao desfeita (rollback). Se h sucesso em todas as aes a informao mantida no banco (commit);


  • Consistncia

    • Deve-se obedecer regras e restries definidas em um banco, como por exemplo uso de chaves estrangeiras ou uso de campos nicos;


  • Isolamento

    • Cada transao deve ser independente de outras transaes;


  • Durabilidade

    • Os resultados de uma transao devem ser permanentes, exceto se outra transao a desfizer.
